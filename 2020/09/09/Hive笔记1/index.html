什么是Hive

使用SQL语言对HDFS数据库进行访问，可以将sql语句转化为mapreduce
drop删除表，会从HDFS上删除数据，meta信息也会删除
误操作、重新创建表

产生背景

MapReduce编程不方便、传统RDBMS人员的需要

HDFS上的文件没有schema的概念
什么是schema：

Hive是构建在Hadoop上的数据仓库、HQL、底层支持多种不同的执行引擎
底层引擎：MR/Tez/Spark
(Apache Hive)[http://hive.apache.org]

统一元数据管理

Hive数据放在HDFS上

元数据信息是放在MySQL中。
元数据：表有什么字段、字段类型是什么，即记录数据的数据。

SQL on Hadoop

Hive、Spark、 SQL、Impala，统一元数据管理，使用Hive创建的元数据，也可以使用Spark或者Impala进行访问。

Hive体系架构：

client： shell、thrift/jdbc(server/jdbc)、WebUI(HUE/Zeppelin)
metastore：
database：name、location、owner
table：name、location、… 从客户端的sql 翻译成 mr任务，到hdfs获取数据。

Hive部署架构

MySql存元数据信息，那MySql坏了怎么办？

1、 MySql主备架构，实现MySql的高可用。

2、VIP：虚拟ip。Hive连接的不是具体的MySQL，哪个没挂掉连哪个。

Hive和RMDBMS 区别

都支持：SQL，包括insert、update等操作。
区别：Hive主要用在离线操作，因为是底层是mr。

Hive Data Definition Language

create、delete、alter

Hive数据结构/抽象

database         HDFS目录
table              HDFS一个目录
data             文件
partition 分区表    HDFS一个目录
bucket 分桶     HDFS一个文件
